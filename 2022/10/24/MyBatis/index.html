映射文件

  • 数据库的每一张表 在Java中对应一个实体类

  • 映射文件

  • <?xml version="1.0" encoding="UTF-8" ?>
<!DOCTYPE mapper
        PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN"
        "http://mybatis.org/dtd/mybatis-3-mapper.dtd">

<mapper namespace="mapper.BrandMapper">

    <!--结果映射
        是为了解决数据库和Java命名规范不同导致没有自动封装
        result标签是用于一般字段的映射，id标签是用于主键字段的映射-->
    <resultMap id="brandResultMap" type="brand">
        <result column="brand_name" property="brandName"/>
        <result column="company_name" property="companyName"/>
    </resultMap>
    <select id="selectAll" resultMap="brandResultMap">
        select * from tb_brand;
    </select>
    <!--占位符 #{} 相当于 ？可以防止SQL注入
              &{} 不可以防止SQL注入-->
    <select id="selectById" resultMap="brandResultMap">
        select * from tb_brand
            where id = #{id};
    </select>
</mapper>

注意事项

在XML文件使用特殊字符可使用快捷键CD生成CD区，在里面的特殊字符就被当做纯文本

参数问题

  • 单个参数：

占位符里是什么无所谓，但建议写清楚

  • 多个参数：

    • 底层自动封装参数Map：1：以arg0 arg1 为键 值为参数 2：以param1 param2为键

    • 手动封装Map：接口方法参数为Map，同样占位符为键

    • 实体类类型封装：占位符为类的属性名（判断属性是否存在是看其存不存在get set 方法）

    • @param(“”):以param的值为键 参数为值 或 param1 param2为键

缓存

一级缓存 (只针对查询)

默认开启，范围是同一个SqlSession

失效的四种情况

  • 不同的SqlSession对象
  • 相同的SqlSession对象但查询条件不同
  • 在俩次相同查询之间有过增删改
  • 手动清除缓存 sqlSession.clearCache()

二级缓存

针对整个SqlSessionFactor

前提

  • 在映射文件中加入标签
  • 实体类 实现 Serializable接口
  • 二级缓存在手动提交之后或者关闭SqlSession之后生效

缓存顺序

先在二级缓存里找，再一级缓存，再数据库

SqlSession关闭后，一级缓存会写到二级缓存里
